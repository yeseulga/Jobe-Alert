# 추가 획득 필요 스킬 및 전략 (Skills to Acquire)

이 문서는 시장 분석을 바탕으로 예슬 님이 5년 차 AI/ML 엔지니어 스펙에 도달하기 위해 추가로 확보하거나 보완해야 할 스킬과 전략을 정리한 문서입니다. 대화를 통해 지속적으로 업데이트됩니다.

## 1. 현재 확보된 역량 (Current Strengths)
- **언어 및 도구**: Python
- **AI/LLM**: OpenAI API, Claude API, LangChain/LangGraph, RAG 구축 경험
- **자동화**: BeautifulSoup, Playwright를 활용한 스크래핑 및 파이프라인 자동화 (launchd)
- **데이터베이스**: SQLite

## 2. 보완이 필요한 핵심 스킬 (Skills to Acquire)

시장 기준(5년 차)과 비교했을 때, 현재 중점적으로 채워야 할 기술적 갭(Gap)입니다.

### 🎯 Phase 1: 백엔드 서빙 및 클라우드 배포 기초 (우선순위: 높음)
- **목표**: 로컬에서 동작하는 자동화 코드를 클라우드 서버에 배포하고, 누구나 쓸 수 있는 API 형태로 서빙할 수 있어야 합니다.
- **필요 스킬**: 
  - `Docker`: 내 컴퓨터가 아닌 어디서든 실행될 수 있도록 환경 컨테이너화.
  - `AWS (EC2)` 또는 `GCP`: 클라우드 가상 환경에 스크립트 띄우기.
- **제안 전략**: 현재 로컬 `launchd`로 도는 `job-alert` 시스템을 FastAPI로 감싸고 Docker로 패키징하여 AWS EC2 프리티어에 배포해보기.

### 🎯 Phase 2: MLOps 및 자동화 고도화 (우선순위: 보통)
- **목표**: 개인용 스케줄러를 넘어, 팀 단위의 협업과 코드 배포 파이프라인 구축.
- **필요 스킬**:
  - `Vector DB` (Chroma, FAISS 등): 단순 API 호출을 넘어 자체 RAG 시스템의 검색 성능 고도화.
  - `PostgreSQL/MySQL`: SQLite를 넘어선 상용 RDBMS 사용 경험.

### 🎯 Phase 3: 정량적 성과 측정 및 시스템 최적화 (마인드셋)
- **목표**: "만들었다"에서 끝나는 것이 아니라 수치화된 성과 도출.
- **필요 역량**:
  - 기존 수동 검색 방식 대비 비용/시간 절감률 계산.
  - LLM API 토큰 최적화, 프롬프트 엔지니어링을 통한 비용 관리 및 속도 개선.
