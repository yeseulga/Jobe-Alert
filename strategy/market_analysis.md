# 5년 차 AI/ML 엔지니어 시장 분석 (Market Analysis)

이 문서는 AI/ML 엔지니어 및 AI 응용 개발자 포지션의 5년 차 시장 요구사항을 정리한 문서입니다. 대화를 진행하며 계속해서 업데이트됩니다.

## 1. 핵심 역량 (Core Competencies)
- **MLOps 및 파이프라인 구축**: 모델의 로컬 개발을 넘어, 학습-평가-배포 전 과정을 자동화(CI/CD)하는 능력.
- **시스템 아키텍처 및 서빙**: 모델을 실제 서비스에 안정적으로 배포하고 대용량 트래픽을 처리하는 능력.
- **비즈니스 문제 해결**: 비즈니스 요구사항을 AI 문제로 정의하고, 모델 도입으로 인한 정량적 성과(비용 절감, 속도 향상, 매출 증대 등)를 도출하는 역량.
- **최신 트렌드 적용**: LLM, RAG, 에이전트 체인(Agentic AI) 등 최신 AI 기술을 실무에 적용하고 평가(Evals)하는 능력.

## 2. 필수 기술 스택 (Tech Stack)
- **언어**: Python (고도화 및 성능 최적화), C++/Go (인프라/서빙 최적화 필요 시)
- **프레임워크**: PyTorch, TensorFlow, Hugging Face
- **인프라 및 배포**: Docker, Kubernetes, AWS/GCP, vLLM/Triton Inference Server
- **백엔드 및 데이터**: FastAPI/Flask, SQL, Vector Database (Pinecone, Milvus 등)
- **MLOps 툴**: LangChain, MLflow, Weights & Biases, GitHub Actions

## 3. 면접 및 이력서 핵심 포인트 (5년 차 기준)
- "어떤 모델을 써봤다"가 아닌 "왜 그 모델을 선택했고, 어떻게 최적화하여 어떤 비즈니스 가치를 만들었는가"를 논리적으로 설명.
- 시스템 전체(데이터 수집 -> 전처리 -> 학습 -> 서빙 -> 모니터링)를 조망할 수 있는 아키텍처 시각 어필.
