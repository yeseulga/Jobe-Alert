# Job-Alert → Job-Application 전환 설계 문서
# office-hours 세션 결과물 — 2026-05-04

---

## 핵심 문제 재정의

현재 시스템 이름: **job-alert** (알람 발송)  
실제로 필요한 시스템: **job-application** (지원서 발송)

공고 알림은 이미 작동한다. 병목은 두 곳에 있다:

| 병목 | 증상 | 원인 |
|------|------|------|
| 회사 판단 불가 | 공고 읽고 "나중에 보자" → 안 봄 | JD 외 정보 없음 (연봉범위, 팀규모, 문화, 성장성) |
| 자소서 장벽 | 지원하고 싶어도 못 함 | 자소서 한 개 = 하루 이상 → 에너지 소진 |

---

## 목표 상태

이메일 하나를 받으면 **30분 안에 지원서를 낼 수 있어야 한다**.

```
[공고 수신]
    ↓
[회사 요약 카드] — 연봉범위, 팀규모, 최근 펀딩, 직원 리뷰 요약
    ↓
[자소서 초안] — 이미 생성되어 있음 (JD + profile/me.md + cv/experiences.md 기반)
    ↓
[사용자 할 일] — 읽기 → 복사 → 수정 20분 → 제출
```

---

## 구현 로드맵

### Phase 1 — 자소서 자동 생성 (가장 임팩트 큰 것 먼저)

**목표**: 이메일에 자소서 초안이 붙어서 옴.

**방법**:
1. `main.py`의 이메일 발송 직전, 각 공고에 대해 자소서 초안 생성
2. `cv/experiences.md` + `profile/me.md` + JD 내용 → LLM API (Claude/GPT) → STAR 포맷 초안
3. 이메일 본문에 공고 + 초안 같이 첨부

**필요한 것**:
- `core/cv_generator.py`: LLM 호출, 자소서 생성
- `ANTHROPIC_API_KEY` 또는 `OPENAI_API_KEY` 환경변수
- `cv/experiences.md` 내용 채우기 (사용자 직접)
- `profile/me.md` 내용 채우기 (사용자 직접)

**판단 기준**: 이 기능만 있어도 지원율 10배 오를 것.

---

### Phase 2 — 회사 인텔리전스 카드

**목표**: 공고별로 회사 요약 정보 자동 첨부.

**방법**:
1. 공고 수집 시 회사명 추출
2. 다음 소스에서 자동 수집:
   - Crunchbase API / 기술 블로그 RSS
   - Glassdoor (웹스크래핑)
   - 최근 뉴스 (Google News RSS)
   - 채용 공고 수 트렌드 (성장 신호)
3. 이메일에 회사별 요약 카드 첨부

**출력 형식 (회사 카드)**:
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🏢 업스테이지 (Upstage)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📍 트랙: A (AI 자체 솔루션) | 우선순위: 2
💰 예상 연봉: 4,000~5,500만 (시리즈B, 80명)
📈 성장: 시리즈B 170억 (2024) | HuggingFace 리더보드 상위
👥 규모: 80명 | 서울 강남
⭐ 글래스도어: 4.2 / 5 (워라밸 3.8)
📰 최근: "Solar LLM API 출시, 글로벌 진출"
```

---

### Phase 3 — 원클릭 지원 트래킹

**목표**: 지원한 회사/공고를 DB에 자동 기록, 팔로업 리마인더.

**방법**:
1. `target_companies.yaml`의 `status` 필드 활용
2. 이메일 내 각 공고에 "지원했음" 체크 링크 (간단한 로컬 API)
3. 마감 3일 전 리마인더

## 즉시 시작할 것

**지금 당장 사용자가 해야 할 것 (코드 없음):**
1. `profile/me.md` 채우기 — 이름, 학력, 기술스택, 목표 한 줄
2. `cv/experiences.md` 채우기 — 경험 3개 이상 (키워드라도 OK)

**그 다음 단계 (Claude와 같이):**
1. `core/cv_generator.py` 구현 — LLM으로 자소서 초안 생성
2. `core/mailer.py` 수정 — 초안을 이메일 본문에 포함
3. 회사 인텔리전스 카드는 Phase 2로
